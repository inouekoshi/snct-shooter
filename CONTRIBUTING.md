# コントリビューションガイド

本プロジェクト（暇つぶしシューティングゲーム）に関心を持っていただきありがとうございます！
コミュニティの皆様からのフィードバックやコードの貢献はいつでも歓迎しています。

以下にプロジェクトに参加するための基本的なプロセスを記載します。

## プロジェクトの目的
このプロジェクトは、スマートフォンブラウザで「サクッと遊べる軽量なゲーム」を提供することを目的としています。
機能追加については、「シンプルさ」「片手操作のしやすさ」「軽量な動作」を損なわない範囲での実装を検討してください。

## 開発を始めるにあたって

プロジェクトの全体像や技術スタックの理解については、以下のドキュメントをお読みください。
- [README.md](README.md): プロジェクト概要とセットアップ
- [docs/spec.md](docs/spec.md): ゲームの仕様に関する詳細
- [docs/architecture.md](docs/architecture.md): 設計やシステム構造に関する解説

## Issues について
バグの報告や機能提案は、[GitHub Issues](../../issues) からお願いします。

### バグを報告する場合
- 発生した環境 (OSのバージョン、ブラウザの種類など) を明記してください。
- 再現手順をなるべく詳細に記載してください。
- 可能であればスクリーンショットやエラーログを添付してください。

### 機能を提案する場合
- なぜその機能が必要なのか、どのようなメリットがあるのかを記述してください。
- インターフェースやデザインに変更がある場合は、参考イメージも頂けると助かります。

## Pull Requests (PR) について
コードを修正・追加して送信する場合は、以下の手順に従ってください。

1. **フォークしてブランチを作成する**
   - リポジトリをフォークし、独自のワークブランチを切ってください。
   - ブランチ名は `feature/xxx`, `fix/xxx` のような分かりやすいプレフィックスを使用してください。

2. **コードを書く**
   - 仕様書やアーキテクチャのガイドラインから外れないようにコーディングします。
   - TypeScriptの型定義、Lintエラーの解消を確認してください。
   - コメントは最小限にし、変数名や関数名で意図を伝えるコーディングを推奨します。

3. **ローカルでテストする**
   - `npm run dev` でローカルサーバーを立ち上げ、スマートフォン実機（またはシミュレータ）で正常に動作し、レイアウトが崩れていないか確認します。
   - `npm run build` を実行してビルドが成功するかどうかも確認してください。

4. **Pull Request を作成する**
   - 本リポジトリの `main` ブランチに向けてPRを作成します。
   - PRの概要に変更点や対応したIssue番号を記載してください。
