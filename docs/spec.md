# 暇つぶしシューティングゲーム 仕様書

> バージョン: v0.1.0  
> 最終更新: 2026-05-05

---

## 1. プロジェクト概要

スマートフォンのブラウザで動作する縦スクロール型シューティングゲーム。  
オフラインでも遊べ、1セッション約15分の暇つぶしを目的とする。  
Chromeの恐竜ゲームのように、軽量・シンプルで誰でも即プレイ可能。

---

## 2. ゲーム仕様

### 2.1 基本ルール

- プレイヤーは画面下部に配置された自機を操作する
- 敵は画面上部から出現し、下方向に移動する
- 時間経過とともに難易度が上昇する（敵の数・速度・攻撃頻度）
- 残機が0になるとゲームオーバー

### 2.2 操作方法

| 操作 | アクション |
|---|---|
| 画面タップ | 弾を発射 |
| 画面スワイプ / タップ移動 | 自機の移動 |

- 移動とタップは独立して処理（片手でプレイ可能な設計を目指す）
- 画面を長押し or ドラッグで自機を動かし、タップで発射

### 2.3 自機

- 残機: 3（ライフ制）
- 敵または敵弾に接触すると残機 -1、一定時間無敵
- 残機 0 でゲームオーバー

### 2.4 敵の種類

| 種類 | 説明 |
|---|---|
| 雑魚敵（通常） | 直進するだけ、弾なし |
| 攻撃敵 | 直進しながらプレイヤーに向けて弾を撃つ |
| ボス | 定期的に出現、大量のHPを持ち複数の弾幕パターンを持つ |

### 2.5 ボス戦

- 一定スコアまたは一定時間ごとにボスが出現
- ボスはHPゲージを持つ
- ボス出現中は雑魚敵の出現を停止
- ボスを倒すとスコアボーナス + 次のステージへ移行

### 2.6 難易度カーブ

| 経過時間 | 変化内容 |
|---|---|
| 0〜2分 | 雑魚敵（通常）のみ、低速 |
| 2〜5分 | 攻撃敵が出現、敵の速度増加 |
| 5分〜 | 攻撃敵の割合増加、弾の速度・頻度増加 |
| 一定間隔 | ボス出現 |

### 2.7 スコア

| イベント | 加算スコア |
|---|---|
| 雑魚敵（通常）撃破 | +10 |
| 攻撃敵撃破 | +30 |
| ボス撃破 | +500 |

- ハイスコアは `localStorage` に保存
- ゲームオーバー時に現在スコアとハイスコアを表示

### 2.8 画面構成

| 画面 | 内容 |
|---|---|
| スタート画面 | タイトル、ハイスコア表示、スタートボタン |
| ゲーム画面 | Canvas描画、スコア・残機・経過時間のHUD |
| ゲームオーバー画面 | スコア、ハイスコア、リトライボタン |

## 3. 非機能要件

| 項目 | 要件 |
|---|---|
| 動作環境 | スマートフォンブラウザ（iOS Safari / Android Chrome） |
| 画面向き | Portrait（縦）固定 |
| フレームレート | 60fps目標 |
| オフライン | PWA（Service Worker）によりオフラインプレイ可能 |
| サウンド | なし |
| パワーアップ | なし |
| データベース | 不要（ローカルのみ） |

| 役割 | 技術 |
|---|---|

## 5. アーキテクチャ

### 5.1 ディレクトリ構成

```
src/
  app/
    page.tsx              # スタート画面
    game/
      page.tsx            # ゲーム画面
    layout.tsx            # PWA設定・メタタグ
  components/
    GameCanvas.tsx        # Canvasをラップするコンポーネント
    HUD.tsx               # スコア・残機表示
  lib/
    game/
      engine.ts           # ゲームループ（requestAnimationFrame）
      player.ts           # 自機クラス
      enemy.ts            # 敵クラス（通常・攻撃・ボス）
      bullet.ts           # 弾クラス
      collision.ts        # 衝突判定ロジック
      score.ts            # スコア管理・localStorage
      difficulty.ts       # 難易度カーブ管理
  public/
    manifest.json         # PWAマニフェスト
    sw.js                 # Service Worker
```

### 5.2 ゲームループ

```
requestAnimationFrame ループ
  ├── 入力処理（Touch Events）
  ├── 状態更新（移動・弾・衝突判定・難易度）
  └── 描画（Canvas clear → 背景 → 敵 → 自機 → 弾 → HUD）
```

## 6. デプロイ
